## Codebase Patterns

- `src/crawl.ts` now centralizes crawl output through a tiny `logEvent(level, event, fields)` helper that renders `[LEVEL] event.name key=value` lines and routes them to exactly one console call per event (`console.log`, `console.warn`, or `console.error`).
- Logging tests capture console output by temporarily replacing `console.log`, `console.warn`, and `console.error`, storing the raw argument arrays, and asserting against `args[0]` substrings rather than exact full-line matches.
- Markdown guess-family observability in `src/crawl.ts` is tracked per seed via a `guessGroups` map keyed by original URL plus `PendingUrl.guessGroupOriginalUrl`; emit exactly one grouped result only after every attempt in that family has a terminal outcome.
- Plain `node --test` for `.ts` crawler tests relies on one-line `src/*.js` bridge modules that re-export the TypeScript sources behind existing `.js` import specifiers; keep those bridge filenames aligned with the `.ts` modules and ignored by ESLint.
