## Codebase Patterns

- Typecheck uses `tsconfig.node.json` (not `tsconfig.json`). The `npm run typecheck` script references the wrong filename and fails; use `npx tsc --project tsconfig.node.json` directly.
- Tests use Node's built-in test runner: `node --import tsx --test test/*.test.ts`
- `npm run lint:js` has pre-existing errors in `AGENTS.md` and `src/rewrite-links.ts`. Use `npx eslint <specific-file>` to check only your files.
- The codebase uses `eslint --fix` for auto-fixable issues. Non-auto-fixable ones need manual optional-chaining rewrites.
- Exports from `src/crawl.ts` are tested via `test/crawl-metadata.test.ts` (unit) and `test/crawl.test.ts` (integration).
- The linter requires commas (not semicolons) as member delimiters in inline type literals: `{ a?: boolean, b?: string }` not `{ a?: boolean; b?: string }`.
- The linter forbids `Array<T>` syntax -- use `T[]` instead (`@typescript-eslint/array-type` rule).
