## 2026-03-27 - US-002

- Refactored `crawlGroup` to accept a `QueueManager` parameter and use `queueManager.fetch()` instead of calling `fetchWithRedirects` directly
- Replaced sequential BFS loop with concurrent fetch model: `submitPending()` submits all pending URLs as non-blocking promises, `processResult()` handles each result and enqueues newly discovered URLs
- Rate-limited (429) handling now uses `queueManager.pauseDomain()` instead of explicit `sleep()`
- `crawl()` creates a `QueueManager` instance and passes it to `crawlGroup`
- Added integration test verifying concurrent in-flight requests (maxInFlight > 1)
- Adjusted US-006 test assertions: with concurrent fetching, redirect-caused duplicate saves have non-deterministic ordering, so new/unchanged counts are now ranges
- Files changed: `src/crawl.ts` (modified), `test/crawl.test.ts` (modified)
- **Learnings for future iterations:**

  - The concurrent `crawlGroup` uses a `Set<Promise<void>>` for in-flight tracking with `Promise.race` to wait. `submitPending()` is called both initially and after each result is processed (inside `.then()`) to immediately submit newly discovered URLs.
  - With concurrent fetching, redirect-caused duplicate fetches (e.g., `/redirect` and `/page-b` both resolving to the same `finalUrl`) may save the content twice before either completes. The `saveContent` "new" vs "unchanged" result becomes non-deterministic. Tests should use range assertions for these counts.

- **Learnings for future iterations:**

  - `parseArgs` from `node:util` requires `strict: false` when you want to ignore unknown flags (like positional args or flags from other tools). Without it, unknown flags throw.
  - ESLint `@stylistic/quote-props` requires unquoted property keys when they don't need quotes (e.g., `diff` and `concurrency` must be unquoted, but `"show-diff"` needs quotes due to the hyphen). Similarly, `@typescript-eslint/dot-notation` prefers `values.concurrency` over `values["concurrency"]`.
